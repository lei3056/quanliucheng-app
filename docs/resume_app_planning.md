# “简历生成”界面深度解析与移动端业务规划文档

## 1. 字段关联与逻辑梳理 (基于公职/企事业单位标准表单)
在深入分析体制内、央国企的简历录入需求时，我们发现信息的颗粒度和严谨度远高于普通的互联网社招。为了适应移动端碎片化、触屏友好的操作习惯，我们将传统 PC 端“整页平铺、多达数十个输入框”的长表单进行降维重组，拆分为**“主干档案”**与**“分支档案”**。

### 1.1 一级维度的确立 (按业务模块划分)
为了避免移动端出现“信息过载”，我们并未弱化外语、证书、校园经历等重要信息，而是将其**合并/归类**，最终将数十个字段收拢为 6 个核心卡片作为一级入口：
1. **[基础身份]**: 底层数据，决定报考资格（年龄、政治面貌）。
2. **[教育经历]**: 强硬性门槛（双一流、专业大类、学历层次）。
3. **[实践与工作经历]**: 能力体现。囊括了**校园学生会/社团工作**、**校园大赛/大创项目**、**体制外社会实践/实习**及正式工作经历。统一采用 STAR 法则结构化。
4. **[技能与证书]**: 显性硬通货。专门用于结构化录入**外语水平（四六级/雅思）**、**职业资格证书（法考/CPA等）**和计算机等级。这些在招考大表中常作为强制过滤条件。
5. **[家庭与社会关系]**: 政审与回避性质的特殊必填项（公职类强关联）。
6. **[综合评价与表彰]**: 主观评价与荣誉归档。将**校园奖励（国奖/省优大等）**与自我评价融合，塑造个人立体画像。

### 1.2 二级维度的确立 (字段优先级与表单化细节)
在每个模块内部，根据招考职位表（Excel）的常见硬性要求，划分为不同的结构：

*   **基础身份 (Basic)**: 姓名、身份证号（反算年龄）、政治面貌等。
*   **教育经历 (Education)**: 起止时间、学校、专业、学历学位。
*   **实践与工作经历 (Experience) - 动态数组**:
    *   **分类**: 选项区分 [全职工作] / [实习] / [校园干部] / [比赛项目]。
    *   **核心**: 单位/赛事、职务/角色、描述。
*   **技能与证书 (Certifications) - 标签+表单**:
    *   **外语**: 语种、等级（CET-4/6、专四/专八）、分数。
    *   **证书**: 统一通过搜索库模糊匹配添加（如“法律职业资格证书 A类”）。
*   **家庭关系 (Family) - 动态数组**: 称谓、姓名、政治面貌、工作单位及职务。
*   **评价与表彰 (Awards & Evaluation)**: 荣誉奖项（按级别分类）、个人自评。

## 2. 移动端“一级/二级”业务交互流程规划
为了避免用户在手机上直接面对海量表单产生的“畏难/弃填”情绪，我们采用**“渐进式表单展示 (Progressive Disclosure)”**。

### 2.1 一级界面：档案概览枢纽 (Profile Hub)
*   **结构设计**: 顶部采用仪表盘/图表设计，直观展示当前简历的“综合竞争力”及“资料完成度”（如 85%）。下方则是 5 大模块的入口菜单。
*   **状态反馈机制**: 每个子模块自带状态追踪徽章。
    *   🟢 **已完善 (Complete)**: 信息完整。
    *   🟡 **待完善/警告 (Warning)**: 缺失关键条目（如缺少一条本科及以上教育经历）。
    *   ⚪ **未填写 (Pending)**: 全新状态。

### 2.2 二级界面：沉浸式多态编辑器 (Immersive Editor)
当用户点击任意一级模块时，右侧/底部将拉起沉浸式二级页面。
*   **复杂表单拆解**: 针对教育、经历、家庭等需要填写多条信息的模块，放弃长表单，采用“实体信息卡片展陈 + 固定底部的新建按钮”逻辑。
*   **AI 赋能输入**: 在自我评价或经历描述环节，引入 AI 标签与重塑引擎，将用户的简短输入（如“干过新媒体”）自动扩写为标准化的公文/专业化描述。
*   **数据联动**: 例如录入身份证号后，实时自动生成并锁定出生年月与性别框，减少手动输入。
